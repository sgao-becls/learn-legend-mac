# 在本程序的基础上做修改版简明指南

要做新的修改版，必须熟悉 Lua 语言。不熟悉也没关系，Lua 很容易学习，只要你学过任何一门编程语言，就可以很容易上手 Lua。

## 1. 修改前的准备

游戏目录下的 `script` 目录保存着所有 Lua 代码，以及 Lua 独立编译器（也不是 Lua 运行必须的，因为可执行程序自身可以实现对 Lua 的编译，但在编写代码后先用独立编译器编译排错比较好）和 Lua 解释器（可独立运行 Lua 程序）。因此并不需要准备特别的东西，当然需要一个文本编辑器。原文推荐 EditPlus（语法高亮、可自定义快捷键编译等）。

建议找些资料学习 Lua。项目中可提供从网上下载的文档和英文参考手册。

另外可通读 `jymain.lua` 代码（原文多处写作 jymian），理解语法和游戏逻辑；代码中有很多注释。

也可以自己编写小代码，用 `lua5.1.exe` 运行以检验效果。

## 3. `script` 目录下的 Lua 文件列表

| 文件 | 说明 |
|------|------|
| `jymain.lua` | 游戏主文件，实现原版游戏功能 |
| `jyconst.lua` | 游戏常量定义 |
| `jymodify.lua` | 修改版本文件，对原版功能的扩充 |
| `old_kdef.lua` | 原版 kdef 生成的 Lua；生成方法参见 `ConvertKdef&talk.txt` |

| `luac5.1.exe` | Lua 编译器 |
| `lua5.1.exe` | Lua 独立运行程序 |

## 4. 游戏调用流程

1. C 程序 `jylua.exe` 进行初始化。  
2. C 程序加载 `script\jymain.lua`。  
3. C 程序运行 Lua 中的主入口函数，控制权交给 Lua。  
4. 主流程加载其他 Lua 文件，按游戏逻辑运行。

## 5. 修改简要流程

1. 用文本编辑器修改代码。  
2. 用 `luac5.1.exe` 编译，排除语法错误。  
3. 运行 `JYLua.exe` 测试功能。  
4. 若程序直接退出，到主目录查看 `debug.txt` 定位错误。

## 6. 修改要点

1. **尽量少改 `jymain.lua`**，尽量把修改放在 `jymodify.lua`，可用函数替换法；也可新增扩充文件，在 `IncludeFile` 中导入。

2. **全局变量限制**：自己使用的变量须声明为局部变量。游戏数据集中在三个表：  
   - **`CC`**：常量等不变数据。  
   - **`JY`**：主要变量与控制数据，含 R*/D*/S*。  
   - **`WAR`**：战斗数据。  
   R* 常用且已格式转换，可用中文键访问，例如 `JY.Person[0]["生命"]` 为主角生命。D*、S* 为二进制，通过 `GetD` / `SetD` / `GetS` / `SetS` 访问。需要新“全程”数据时酌情放入上述三表，勿新增裸全局变量。

3. 实现新功能时先查 `jymain.lua` 是否有类似函数，尽量组合复用。

4. **物品修改**：参考 `jymodify.lua` 中的例子；新物品类型常设为药品以便平时与战斗可选用；属性若需避免被自动战斗当药吃掉，需按文中所述把相关属性设为 0 等（见原文细节）。

5. **场景事件**：旧方式仍保留（D* 三个事件编号对应函数）；新场景建议每场景一个主事件入口、每 D* 一个处理函数，按状态改事件编号并在处理函数分支。多个关联场景可合并到一个 Lua 文件。`jymodify.lua` 中有河洛客栈新方式示例。

6. 固定数据多在 **`jyconst.lua`**，可按需修改。

7. **离队人员事件**：在 `jyconst.lua` 中设置每人离队时调用的函数。

8. 游戏中禁止使用未约束的全局变量，否则易出错。

9. **繁体 Big5 环境**：  
   - 源代码字符集：繁体下开发可能需转 Big5，并设置 `CC.SrcCharSet`。  
   - 操作系统字符集：可设 `CC.OSCharSet`。  
   - 字体：可设 `CC.FontName`。  
   以上均在 `jyconst.lua` 中配置。

10. **编译后运行**：可用 `luac5.1.exe` 编译为字节码，改名为 `.lua` 使用（先备份）。命令示例：  
    `luac5.1 -o outputfile sourcefile`  
    若发布字节码修改版，建议一段时间后也发布源码供他人参考。

## 7. 常见问题

### a. 刚运行就退出，无错误提示

多为 Lua 语法错误：查看 `debug.txt`，或用 `luac5.1 sourcefile` 在命令行编译查看错误。

### b. 运行中间退出，无提示

多为未赋值变量或误用全局变量、拼写错误；同样可查 `debug.txt`。

### c. 出现 Windows 级错误

多为传给 C API 的参数不合法。可用 `lib.Debug`（原文写 `lib.debug`，以 API 文档为准）配合 `string.format` 输出调试信息。

## 8. 一些关于修改的思路

- 少改 `jymain` 结构，用新函数扩展。  
- 物品与场景事件已有挂接点；其他扩展可仿照在关键路径上判断“是否有新处理函数再调用”。  
- 需在存档中保存自定义数据时，可在读写进度处插入逻辑，或利用 R* 新 idx、空武功/物品数据位等；新状态放在 `JY` 等表中，勿新增全局变量。
